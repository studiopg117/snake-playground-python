# snake-playground Project (Python + Pygame)

> Follow this specification strictly. Do not ignore any constraints.

## 1. 專案目標

請在目前這個專案中建立一個「可實際遊玩的 Python 貪吃蛇遊戲」。

---

## 2. 環境與工具限制（最高優先）

### 必須遵守

- 必須使用 pipenv 作為虛擬環境與套件管理工具
- 所有套件安裝必須透過 pipenv install

### 禁止

- ❌ 使用 venv
- ❌ 使用 pip install（除非透過 pipenv）

---

## 3. 技術要求

- 使用 Python
- 使用 pygame
- 使用 pipenv 管理虛擬環境
- 以 main.py 作為主程式入口

---

## 4. 專案輸出

### 請建立以下檔案

- main.py（遊戲主程式）
- Pipfile（由 pipenv 自動產生）
- requirements.txt
- README.md
- data/high_scores.json（前五名排行榜資料檔，需保存玩家名稱與分數，若不存在可由程式自動建立）
- assets/（可選，用於封面圖片或選單視覺素材）

## 5. 功能需求

### 請實作以下遊戲功能

#### 核心玩法

- 使用方向鍵控制蛇移動
- 蛇吃到食物後身體變長
- 蛇撞到牆時，不會死亡，需從畫面另一側穿出（wrap-around）
- 分數、等級、速度需採用一致聯動機制
- 等級不得直接依據目前分數計算，避免後期因單次加分提高而導致等級跳升過快
- 等級需依據「吃到的食物數量」提升
- 每吃到 5 個食物提升 1 個等級
- Level 1 為遊戲起始等級
- Level 1 時，每吃到一個食物加 1 分
- Level 2 時，每吃到一個食物加 2 分
- Level 3 時，每吃到一個食物加 3 分
- 以此類推
- 遊戲速度需依據目前等級自動調整
- 遊戲初始速度（Level 1）為 6 FPS
- 每提升 1 個等級，遊戲速度增加 1 FPS
- 遊戲速度最高上限為 20 FPS
- 當等級提升時，分數規則與速度需同步提升
- 速度提升必須平滑且可感知，避免難度突然暴增

#### 視窗與版面需求

- 遊戲視窗大小需調整為目前版本的一半
- 遊戲區大小固定為 320 x 320
- HUD 高度需配合縮小後的視窗比例調整，建議為 40
- 視窗總高度需為 HUD 高度 + 遊戲區高度
- 縮小視窗後，蛇、食物、格線、HUD 排版仍需保持清楚可讀，不可過度擁擠

#### 遊戲結束條件

- 撞到自己時遊戲結束

#### UI需求

- 畫面需顯示目前分數
- 畫面需顯示目前等級（Level）
- 不需要顯示目前速度（FPS）
- 畫面頂部需保留獨立的 HUD 資訊列
- HUD 必須位於遊戲區上方，不能覆蓋蛇、食物或格線遊戲區
- 遊戲區需從 HUD 下方開始繪製
- HUD 內容需整齊排列於上方，避免遮擋遊戲畫面
- 主選單、排行榜、遊戲中、遊戲結束需有清楚的畫面狀態切換
- 主選單需支援鍵盤操作
- 主選單需支援滑鼠點擊操作
- 主選單需具備較完整的封面視覺設計，而不只是純文字列表
- 主選單需包含清楚的遊戲標題、背景視覺元素與選單區塊
- 主選單背景可使用程式繪製的視覺效果（例如漸層、裝飾圖形、半透明面板），或使用本地圖片資源
- 若未使用圖片資源，仍需提供足夠的視覺層次，使主選單看起來像封面畫面而非純文字畫面
- 主選單選項在滑鼠 hover 或鍵盤選取時，需有明確的視覺反饋
- 遊戲結束後可重新開始，或返回主選單

#### 主選單與紀錄系統需求

- 遊戲啟動後不得直接進入遊戲，必須先進入主選單（Menu）
- 主選單需作為遊戲封面畫面（Cover Screen）
- 主選單至少需提供以下選項：
  - Start Game
  - High Scores
  - Quit
- 主選單需同時支援鍵盤選擇與滑鼠點擊操作
- 若使用滑鼠操作，主選單項目需有明確可點擊區域
- 選擇 Start Game 後才正式進入遊戲
- 選擇 High Scores 後需進入排行榜畫面
- 排行榜畫面需顯示前五名最高分
- 排行榜每筆資料需包含玩家名稱與分數
- 排行榜畫面需提供返回主選單的方式
- 遊戲結束後，若本次分數進入前五名，需進入名字輸入流程
- 玩家可輸入名稱後寫入排行榜
- 若玩家未輸入名稱，可使用預設名稱，例如 Player
- 排行榜資料需持久化儲存，不可只存在記憶體中
- 排行榜資料需使用本地 JSON 檔案儲存
- 若排行榜資料檔不存在，程式需自動建立預設檔案
- 排行榜資料檔格式需清楚可讀，至少包含 name 與 score 欄位
- 玩家名稱輸入需支援英文字母、數字、空白鍵與 Backspace
- 玩家名稱輸入框需顯示可見的文字游標（caret）
- 文字游標需具備閃爍效果，讓玩家知道目前正在輸入
- 玩家名稱需限制最大長度，建議為 10 個字元
- 玩家名稱可包含空白，但需避免全為空白；若最終輸入內容為空，可使用預設名稱，例如 Player
- 名字輸入畫面需有清楚提示，例如「Enter your name」
- 玩家可按 Enter 確認送出名稱
